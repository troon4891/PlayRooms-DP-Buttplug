### 2. Source file integrity

Verify the copied source files match the originals from the Host repo:

- `src/client.ts`: Should begin with `import { ButtplugClient, ButtplugNodeWebsocketClientConnector, ButtplugClientDevice } from "buttplug";` and export `connectClient`, `disconnectClient`, `startScanning`, `stopScanning`, `sendCommand`, `getDeviceStates`, `getDiscoveredDevices`, `refreshDeviceStates`, `isConnected`, `isScanning`, `onDevicesChanged`, `onDiscoveredChanged`.
- `src/engine.ts`: Should export `startEngine`, `stopEngine`, `restartEngine`, `isEngineRunning`. Should reference `intiface-engine` CLI binary.
- `src/device-approval.ts`: Should export `getOrCreateDevice`, `approveDevice`, `denyDevice`, `resetDevice`, `forgetDevice`, `updateLastSeen`, `getDeviceByIdentifier`, `getDeviceGlobalSettings`, `updateDeviceGlobalSettings`, `getApprovedDevices`, `getAllDeviceRecords`, `isDeviceApproved`, `cleanupStaleDevices`. Should define `DEFAULT_GLOBAL_SETTINGS` with `maxIntensity: 1.0`.
- `src/protocol-filter.ts`: Should define `PROTOCOL_DEVICE_PATTERNS` with at least 15 protocol entries including `lovense`, `hismith`, `wevibe`, `satisfyer`. Should export `matchesEnabledProtocol`, `getProtocols`, `getEnabledProtocols`, `setProtocolEnabled`.
